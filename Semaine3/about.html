3 — Comment connecter le CSS au HTML (3 méthodes)

Il existe trois façons d'ajouter du CSS :

  1. Externe (fichier .css) — recommandé pour la maintenance.
  2. Interne (<style> dans <head>) — utile pour tests ou pages uniques.
  3. Inline (attribut style) — à éviter sauf exception.

3.1 CSS externe — exemple

<link rel="stylesheet" href="style/style.css">
Chemins courants :
  • href="style/style.css" — si votre fichier HTML est à la racine et le CSS dans /style/

  • 1 — Structure de base d’un fichier HTML

    Un document HTML standard commence ainsi : 

    <!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Titre de la page</title>
</head>
<body>
  <!-- contenu ici -->
</body>
</html>
    Explications détaillées :
    • <!DOCTYPE html> — indique au navigateur qu'il s'agit d'un document HTML5. Sans cela, le rendu peut tomber en Quirks Mode (comportement ancien).
    • <html lang="fr"> — la langue est importante pour l'accessibilité (lecteurs d'écran), le SEO, et la correction orthographique dans certains éditeurs.
    • <meta charset="UTF-8"> — définit l'encodage ; UTF-8 supporte tous les caractères (accents, symboles).
    • <meta name="viewport" ...> — indispensable pour que la page soit responsive sur les mobiles.
    Exemples concrets d'erreurs liées à la structure
    • Oublier <meta charset="UTF-8"> → caractères accentués affichés en "Ã©" etc.
    • Placer le <script> bloquant en haut du <head> sans defer → ralentit l'affichage.

    2 — Balises sémantiques : header, nav, section, footer, article, aside

    Les balises sémantiques décrivent le rôle du contenu. Exemple :

    <header>...en-tête du site (logo, menu)</header>
<nav>...navigation principale</nav>
<main>
  <section>...partie du contenu</section>
  <article>...article autonome</article>
  <aside>...contenu secondaire (barre latérale)</aside>
</main>
<footer>...bas de page</footer>
    Pourquoi utiliser ces balises ?
    • Accessibilité : les lecteurs d'écran peuvent naviguer par régions (header, nav, main, footer).
    • SEO : les moteurs comprennent mieux la structure.
      • href="../css/main.css" — remonte d’un dossier puis cherche /css/main.css
    • href="/assets/css/site.css" — chemin absolu depuis la racine du site

    <style>

    3.3 Inline — exemple

    <div style="color:red; padding:8px;">Texte</div>

    Pratique pour tests ou styles dynamiques via JS, mais difficile à maintenir.

  display:flex;
  justify-content:space-between;

    4 — Comment connecter JavaScript au HTML (avec exemples)

    Trois façons principales d'inclure JS :

    1. <script src="script.js"></script> placé à la fin du <body> (traditionnel)
    2. <script src="script.js" defer></script> dans le <head> — script chargé et exécuté après le parsing du HTML
    3. <script src="script.js" async></script> dans le <head> — script exécuté dès qu'il est téléchargé (utile pour scripts tiers non dépendants)

    Exemples :

    // 1) Script en bas du body (simple et sûr)
<body>
  ...
  <script src="js/main.js"></script>
</body>

// 2) Script avec defer (idéal pour la plupart des cas)
<head>
  <script src="js/main.js" defer></script>
</head>

// 3) Script async (pour analytics / pubs)
<head>
  <script src="https://example.com/analytics.js" async></script>
</head>
    Quand utiliser quoi :
    • defer — recommandé pour vos propres scripts qui manipulent le DOM (ils s'exécutent après le parsing).
    • async — bon pour des scripts qui n'ont pas besoin du DOM (ex: analytics, publicités).
    • Placer le script en bas du body est simple et évite la plupart des problèmes de blocage.

    Exemple concret : interagir avec un formulaire de réservation

    <!-- HTML -->
<form id="res-form">
  <input id="name" name="name" required>
  <input id="date" name="date" type="date" required>
  <button type="submit">Réserver</button>
</form>

/* JS (js/main.js) */
document.getElementById('res-form').addEventListener('submit', function(e){
  e.preventDefault(); // empêche l'envoi par défaut
  const name = document.getElementById('name').value;
  const date = document.getElementById('date').value;
  if(!name || !date){ alert('Veuillez remplir tous les champs'); return; }
  // ici vous pourriez envoyer via fetch() vers un backend
  alert('Réservation reçue pour ' + name + ' le ' + date);
});
    Bonnes pratiques JS : Toujours valider côté client et côté serveur. Utilisez fetch() pour les requêtes modernes (AJAX).

    5 — Box model, margin, padding, border — exemples

    Le modèle de boîte : chaque élément HTML est une "boîte" composée de :

    • content — le contenu (texte, image)
    • padding — espace interne
    • border — bordure
    • margin — espace externe (entre éléments)
    /* Exemple : */
.box {
  width:300px;
  padding:20px;
  border:2px solid #ddd;
  margin:16px;
}

    Flexbox — mise en ligne d'éléments

    /* CSS */
.container{ display:flex; gap:12px; align-items:center; justify-content:space-between; }
.item{ flex:1; }

    Flexbox est idéal pour les barres de navigation, header, cartes alignées sur une ligne.

    Grid — mise en grille

    /* CSS */
.grid{ display:grid; grid-template-columns: repeat(3, 1fr); gap:16px; }

    Grid est puissant pour les mises en page complexes (galeries, dashboards).

    Exemple concret : header responsive avec flexbox 
    header{ display:flex; align-items:center; justify-content:space-between; padding:12px 20px; }
.logo{ display:flex; gap:12px; align-items:center; }
nav ul{ display:flex; gap:16px; list-style:none; }

    6 — Spécificité CSS, cascade et ordre

    Quand plusieurs règles s'appliquent, le navigateur choisit celle qui a la spécificité la plus élevée. Ordre de priorité (simplifié) :

    1. Styles inline (ex : style="...") — très spécifiques
    2. ID selectors (#monId)
    3. Class/attribute/pseudo-class selectors (.maClasse / [type="text"] / :hover)
    4. Elements selectors (header, p)
    5. Le dernier règle dans le fichier remporte en cas d'égalité
    /* Exemple */
p { color: black; }          /* faible spécificité */
.text { color: red; }        /* classe */
#special p { color: blue }   /* id + élément => plus spécifique
    Astuce : évitez !important sauf cas extrême — il casse la maintenabilité.

    7 — Images : src, alt, formats et chemins

    Image tag :

    <img src="images/hero.jpg" alt="Photo du restaurant" width="800" height="450">
    Conseils pratiques :
    • alt est essentiel pour l'accessibilité et SEO.
    • Définissez width et height ou utilisez CSS pour éviter le "layout shift".
    • Utilisez WebP/AVIF pour de meilleures performances lorsque possible, avec fallback JPEG/PNG si nécessaire.
    Chemins relatifs vs absolus
    • Relatif : images/logo.png (depuis le fichier HTML)
    • Absolu : /assets/images/logo.png (depuis la racine du site)

    8 — Formulaires & accessibilité (ex: réservation)

    Formulaire accessible :

    <form id="res-form">
  <label for="name">Nom complet</label>
  <input id="name" name="name" type="text" required>

  <label for="date">Date</label>
  <input id="date" name="date" type="date" required>

  <button type="submit">Réserver</button>
</form>
    Explication :
    • <label for="id"> lie le label au contrôle — facilite le focus et les lecteurs d'écran.
    • required active la validation HTML5 basique côté client.
    • Utilisez aussi aria-* si nécessaire pour des widgets complexes.

    Exemple de validation & envoi via JavaScript (fetch)

    // JS (extrait)
document.getElementById('res-form').addEventListener('submit', async function(e){
  e.preventDefault();
  const data = { name: this.name.value, date: this.date.value };
  if(!data.name){ alert('Nom requis'); return; }
  // Exemple : envoi vers /api/reservations (backend requis)
  const resp = await fetch('/api/reservations', {
    method:'POST',
    headers:{'Content-Type':'application/json'},
    body: JSON.stringify(data)
  });
  if(resp.ok) alert('Réservation prise');
});
    Important : Toujours valider côté serveur aussi (sécurité & données fiables).

    10 — Débogage & outils (DevTools)

    Utilisez les DevTools (F12) pour :

    • Inspecter les éléments (HTML/CSS appliqué) — trouvez quelle règle CSS prend le dessus.
    • Console — erreurs JS (ReferenceError, TypeError).
    • Network — vérifier que vos fichiers (CSS/JS/images/fetch) sont bien chargés (200 OK).
    • Performance / Lighthouse — audits de performance, accessibilité, SEO.
    Exemple concret d'erreur courante

    Si style.css ne se charge pas : vérifiez le chemin dans le href, regardez l'onglet Network (404 si fichier introuvable). Si le CSS n'est pas appliqué, vérifiez la spécificité, l'ordre des fichiers et si le cache bloque la mise à jour (Ctrl+F5).

    11 — Exemple complet (mini-page) : HTML + CSS + JS

    Voici un exemple minimal mais complet d'une page "À propos" et comment tout se relie :

    <!-- index.html (extrait) -->
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>À propos – Le Gourmet</title>
  <link rel="stylesheet" href="style/style.css">
</head>
<body>
  <header>
    <div class="logo"><img src="images/gourmetlogo.png" alt="Logo"></div>
    <nav><ul><li><a href="index.html">Accueil</a></li>...</ul></nav>
  </header>

  <section>
    <h2>À propos de nous</h2>
    <p>Le Gourmet est...</p>
  </section>

  <footer><p>&copy; 2025 Le Gourmet</p></footer>

  <script src="js/main.js" defer></script>
</body>
</html>

    Dans style/style.css : toutes les règles de style (header, nav, .hero, footer...).

    Dans js/main.js : interaction (menu burger, validation formulaire, appels API).

    Exemple JS : activer un menu "burger" 
    // js/main.js
document.querySelector('.burger').addEventListener('click', function(){
  document.querySelector('nav ul').classList.toggle('open');
});

    12 — Accessibilité (A11Y) : points importants

    • Utiliser des balises sémantiques (nav, main, header, footer).
    • Toujours fournir alt pour les images.
    • Associer <label for="id"> aux inputs.
    • Contrastes de couleurs suffisants (Lighthouse contrast).
    • Gérer la navigation clavier (focus visible, tabindex si nécessaire).

    13 — Erreurs fréquentes & comment les corriger

    • Fichier CSS non chargé : vérifier chemin, maj du cache, console Network (404).
    • JS : null reference : element not found → script exécuté avant le DOM (utilisez defer ou placez le script en bas).
    • Mauvaise utilisation des sélecteurs : .class vs #id vs tag — vérifier la spécificité.
    • Images trop lourdes : compressez, utilisez WebP/AVIF, lazy-loading : <img loading="lazy" ...>.
